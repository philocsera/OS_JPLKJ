# Week 09 종합 보고서

## 개요

xv6를 각자 학습하고, 학습 결과를 바탕으로 LLM 기반 OS 개발 보조 시스템의 가능성을 탐색하였습니다.

---

## 1 xv6 현재 구조의 한계

| 영역 | 한계 |
|------|------|
| 스케줄러 | 우선순위 없음, 실행 시간 기록 없음, starvation 판단 기준 없음 |
| 메모리 | Swap 없음, OOM 처리 없음, 메모리 공정성 없음 |
| 시스템 콜 | 21개뿐, 네트워크/스레드 관련 없음 |
| 파일시스템 | 저널링 없음, 성능 최적화 없음 |

---

## 2. 참여자별 기여 요약

| 팀원 | 주요 기여 내용 |
|------|----------------|
| 김경선 | xv6 기본 구조 학습 정리 (프로세스, 시스템 콜, 스케줄러, sleep/wakeup) |
| 박규태 | xv6 전체 구조 분석 (4대 영역별 장단점, LLM 적용 관점 분석) |
| 조승찬 | xv6 핵심 개념 요약 (전체 흐름, 인터럽트와 트랩 중심) |
| 조현성 | Claude/Codex 활용 xv6 분석, 자연어 Shell 및 AI 스케줄러 아이디어 도출 |
| 이찬주 | xv6 심층 학습 (proc.h/c, 우선순위 스케줄러 설계), Kgent 논문 기반 검증 심층구조 설계 제안 |
