# LLM 기반 메모리 관리 확장 계획

> xv6-riscv에 LLM이 메모리 정책을 제안하고 커널이 실행하는 구조를 추가한다.
> 기존 `schedhint` (CPU 스케줄링 힌트)의 메모리 버전.

> **2026-05-13 개정 노트**: Phase 3(Swap)의 실제 구현 난점과 코드베이스
> 검증 결과를 반영하여 구조를 재정리했다. 핵심 변경:
> - Phase 1/2를 **필수 구현**, Phase 3(Swap)을 **심화/optional**로 분리
> - `walk()` static 문제 등 Phase 3 진입 시 막히는 항목을 사전 명시
> - `kalloc()` 실패 경로를 동기 evict가 아닌 **사용자 공간 사전 폴링** 방식으로 결정
> - swap 디스크는 방법 A(fs.img 공유) 대신 **방법 B(별도 swap.img)** 권장
> - quota 적용 시점을 lazy/eager 양쪽에서 동작하도록 명시

---

## 1. 전체 방향

```
xv6 커널 (우리가 직접 구현)          호스트 bridge          LLM (Solar)
─────────────────────────────        ──────────────         ───────────
proc_info에 sz 포함                  <<MEM>> 수신           메모리 압박
sys_getmemstat                  →    Solar API 호출    →    프로세스 분석
sys_setmemquota                 ←    <<MEMACT>> 응답   ←    액션 제안
swap 커널 구현 (Phase 3)
```

LLM은 직접 메모리를 건드리지 않는다.
커널이 구현한 syscall만 사용하며, LLM은 정책 힌트만 제공한다.

**3중 안전망**: 커널 가드 + bridge 화이트리스트 + `memhint` 클라이언트의
액션 검증. `schedhint`와 동일한 구조.

## 2. 필요한 이론

### 2-1. xv6 현재 메모리 구조 (기반 지식)

| 파일 | 역할 |
|------|------|
| `kernel/kalloc.c` | 물리 메모리 할당/해제. 4KB 페이지 단위 free list |
| `kernel/vm.c` | 가상 메모리, 페이지 테이블 조작 |
| `kernel/memlayout.h` | 물리 주소 레이아웃 상수 |

- 물리 메모리: QEMU 기본 128MB (`PHYSTOP = 0x88000000`)
- `kalloc()` 실패 시 그냥 0 반환 → 프로세스 죽음
- Swap 없음, OOM 킬러 없음, 메모리 쿼터 없음
- **현재 sbrk는 lazy allocation 기본** (`edit0508.md`의 `sys_sbrk` 참고). 즉
  `sbrk(n)`은 `p->sz`만 키우고 실제 페이지는 `vmfault()`에서 할당. EAGER
  플래그를 명시한 경우에만 `growproc()` → `uvmalloc()`을 즉시 호출.

### 2-2. RISC-V Sv39 페이지 테이블

xv6-riscv는 Sv39 방식: 가상 주소 39비트, 3단계 페이지 테이블.

PTE (Page Table Entry) 구조:
```
[63:10] PPN (Physical Page Number)
[9]     RSW (소프트웨어용)
[8]     D  — Dirty: 페이지가 쓰여진 적 있음 (하드웨어 세팅)
[7]     A  — Accessed: 페이지가 접근된 적 있음 (하드웨어 세팅)
[6]     G  — Global
[5]     U  — User 접근 가능
[4]     X  — Executable
[3]     W  — Writable
[2]     R  — Readable
[1]     V  — Valid: 0이면 이 PTE로 접근 시 페이지 폴트 발생
```

**핵심**: V=0으로 설정하면 해당 페이지 접근 시 trap이 발생한다.
Swap 구현에서 이 비트를 이용해 "이 페이지는 디스크에 있음"을 표시한다.

> ⚠️ **주의**: xv6는 이미 lazy allocation에서 "V=0이지만 사용자 영역에 속한 va"를
> "아직 할당되지 않은 페이지"로 해석한다. 스왑 마커는 lazy 미할당과 구분되어야
> 하며, PPN 자리가 0이 아닌 V=0 PTE만 "스왑된 페이지"로 인식한다.

### 2-3. 페이지 폴트 처리 (Swap의 핵심)

현재 xv6 `kernel/trap.c`의 `usertrap()`은 scause 13/15를 받으면 이미
`vmfault()`를 호출해 lazy 할당을 처리한다. 스왑 추가 후에는 `vmfault()`가
다음 흐름을 가져야 한다.

```
페이지 폴트 (scause 13: load, 15: store)
    ↓
stval에서 폴트 발생 va 읽기
    ↓
walk(pt, va, 0) → pte
    ↓
case 1: pte == 0 또는 *pte == 0 → lazy 미할당 (기존 동작)
        → kalloc + mappages
case 2: pte != 0 && V=0 && (PPN >> 10) != 0 → 스왑된 페이지
        → swapin_page(pt, va, slot)
case 3: 그 외 → 진짜 잘못된 접근 → setkilled(p)
```

RISC-V에서 페이지 폴트 관련 레지스터:
- `scause`: 폴트 종류 (13=load, 15=store, 12=instruction)
- `stval`: 폴트 발생한 가상 주소

### 2-4. 페이지 교체 알고리즘

메모리가 부족할 때 어떤 페이지를 디스크로 내보낼지 결정.

**Clock 알고리즘 (LRU 근사, xv6에 적합)**
```
페이지들을 원형 리스트로 관리
시계 바늘이 돌면서:
  - A비트 = 1 → 0으로 초기화 후 넘어감 (최근 사용됨)
  - A비트 = 0 → 이 페이지를 교체 대상으로 선택 (오래 안 사용됨)
```

RISC-V PTE의 A(Accessed) 비트를 직접 읽을 수 있어서 구현 가능.

### 2-5. Swap 파티션 설계 — 방법 B 권장으로 변경

xv6는 virtio 디스크 하나를 사용한다.
현재: 전체를 파일시스템 (`fs.img`)으로 사용.

Swap 추가 방법:
- ~~**방법 A**: `fs.img` 크기를 늘리고 뒷부분을 swap 영역으로 예약~~
  → fs 슈퍼블록/비트맵 영역과 충돌 위험. 디버깅 시 어느 블록이 fs인지
     swap인지 구분 어려움.
- **방법 B (권장)**: 별도 virtio 디스크 추가 (`-drive file=swap.img,...,id=swap,index=1`)
  → Makefile에서 `dd if=/dev/zero of=swap.img bs=4096 count=1024`로 미리 생성.
     `virtio_disk.c`에서 dev 인자로 분기 (`SWAPDEV=2` 등).

Swap 슬롯 관리:
```c
#define SWAPSLOTS 1024  // 스왑 슬롯 개수 (= 4MB)
uint8 swap_bitmap[SWAPSLOTS];  // 0=free, 1=used
```

PTE에 스왑 슬롯 번호 인코딩 (V=0일 때 PPN 자리 재활용):
```
PTE [V=0, PPN field = swap_slot_number] (PPN field는 비트 10 이상)
```

## 3. 신규 syscall 설계

### 기존 syscall 수정

**`sys_proclist` (SYS_proclist = 26)**
- `struct proc_info`에 `uint64 sz` 필드 추가
- `proclist_fill()`에서 `out[n].sz = p->sz` 한 줄 추가
- 이것만으로 LLM이 프로세스별 메모리 사용량 관찰 가능

### 신규 syscall

| 번호 | 이름 | 인자 | 반환 | Phase | 설명 |
|------|------|------|------|-------|------|
| 27 | `sys_getmemstat` | `struct memstat *` | 0/-1 | 1 | 전체+스왑 메모리 통계 |
| 28 | `sys_setmemquota` | `pid, uint64 bytes` | 0/-1 | 2 | 프로세스 메모리 상한 설정 |
| 29 | `sys_swapout` | `pid` | 0/-1 | 3 | 특정 프로세스 페이지 강제 스왑 |

#### `struct memstat` 정의 (신규, `kernel/memstat.h`)
```c
struct memstat {
  uint64 total;       // 전체 물리 메모리 (bytes)
  uint64 free;        // 현재 여유 메모리
  uint64 swap_total;  // 스왑 총 크기 (Phase 3 전에는 0)
  uint64 swap_used;   // 스왑 사용 중인 크기 (Phase 3 전에는 0)
};
```

#### `sys_setmemquota` 동작 — lazy/eager 양쪽 모두 체크

현재 xv6는 lazy allocation이 기본이므로 `uvmalloc()` 한 곳에만 체크를
넣으면 lazy 경로에서 쿼터가 무시된다. 두 곳에 모두 체크가 필요하다.

```c
// kernel/proc.h의 struct proc에 추가
uint64 mem_quota;  // 0이면 무제한

// kernel/sysproc.c의 sys_sbrk에서 lazy 경로 체크
//   addr + n > p->mem_quota 인 경우 반환 -1
// kernel/vm.c의 uvmalloc()에서 eager 경로 체크
//   if(p->mem_quota && p->sz + n > p->mem_quota) return 0;
// kernel/vm.c의 vmfault()에서 페이지 매핑 직전 한번 더 안전 체크
//   (lazy로 sz는 늘었지만 실제 매핑이 quota를 넘기는 경우)
```

#### `sys_swapout` 동작 (Phase 3)
```
1. pid로 프로세스 찾기 + p->lock 획득
2. p->state 검증 (ZOMBIE/UNUSED 거부)
3. pid 1(init), pid 2(sh) 거부
4. 페이지 테이블 순회 → A비트=0인 페이지 찾기
5. 페이지를 디스크에 쓰고 PTE V=0으로
6. 물리 페이지 kalloc free list에 반환
7. p->lock 해제
```

## 4. `kalloc()` 실패 경로 설계 — 사전 폴링 방식

**원래 안 (보류)**: `kalloc()` 실패 시 커널이 자동 eviction 트리거.
- 문제 1: 자동 evict는 LLM 정책이 끼어들 자리가 없다 (전체 컨셉이 무너짐).
- 문제 2: page fault handler 안에서 `bread/bwrite`를 호출하면 sleeplock
  순서/log transaction 컨텍스트 문제 가능성.

**결정**: 동기 evict 경로를 만들지 않는다. 대신 사용자 공간 `memhint`가
주기적으로 메모리 상태를 폴링하여 사전에 swap을 트리거한다.

```
memhint (사용자 데몬, 예: 1초 주기)
  → getmemstat() + proclist()
  → bridge 전송
  → LLM이 "free < 20%이면 pid X swap" 응답
  → swapout(X) 호출 → 페이지 미리 회수

kalloc()은 기존 동작 유지 (실패 시 0 반환).
```

이 구조의 장점:
- 커널 코드 변경 최소화 (`kalloc.c`는 사실상 그대로)
- LLM 응답 latency가 page fault 경로에 들어오지 않음
- 디버깅 시 메모리 상태 변화를 사용자 공간에서 추적 가능

단점:
- 갑작스러운 burst allocation은 막을 수 없음 (memhint 폴링 주기보다 빠른 경우)
- → quota로 보완 (Phase 2에서 충분)

### Phase 3 (Optional, Swap) — 위험 요소 사전 해결 필요

**선결 조건 (이거 안 풀고 시작하면 막힘)**:

1. **`walk()` export 문제**: `kernel/vm.c`의 `walk()`는 함수 정의 앞에
   `static`이 붙어 있어 외부 파일(`swap.c`)에서 직접 호출 불가.
   해결책 둘 중 선택:
   - (A) `vm.c`에서 `walk` 앞의 `static` 제거 + `defs.h`에 prototype 추가
   - (B) `vm.c`에 `pte_t *walk_pub(pagetable_t, uint64)` 래퍼 export
2. **`uvmcopy()` 호환성**: fork 시 부모 페이지 테이블을 자식에게 복사하는데,
   현재 코드는 V=0 PTE를 만나면 panic 또는 skip한다. swap된 페이지가 있는
   상태로 fork하면 자식이 페이지를 잃거나 커널 panic. → `uvmcopy()`에서
   V=0이지만 swap 마커가 있는 PTE는 swap_alloc 새로 받아 디스크에 복제
   하거나, 부모 swap 슬롯에 refcount 추가.
3. **`uvmunmap()` 호환성**: 프로세스 종료 시 swap 슬롯도 함께 해제해야 함.
   `freeproc` → `proc_freepagetable` → `uvmunmap` 경로에서 V=0 PTE의 PPN
   자리가 swap slot이면 `swap_free()` 호출 추가.
4. **`freewalk()` 호환성**: 페이지 테이블 자체를 해제할 때 leaf PTE가 모두
   0인지 확인하는데, swap 마커가 들어있으면 panic. unmap 단계에서 PTE를
   완전히 0으로 만들어야 함.
5. **swap.img 위치**: 방법 B 사용. `Makefile`의 qemu 타겟에 `-drive
   file=swap.img,...,index=1` 추가, `mkfs/mkfs` 변경 없음.
6. **`bread/bwrite` 컨텍스트**: swapin은 `vmfault` (user trap) 안에서
   호출되며, log transaction을 시작/종료하지 않는다. swap I/O는 fs log를
   거치지 않으므로 `bread/bwrite`를 직접 호출해도 안전하지만, 별도 디바이스
   (SWAPDEV)를 써야 fs.img의 buffer cache 경쟁을 피할 수 있음.

| 파일 | 변경 내용 |
|------|----------|
| `kernel/swap.h` | **신규**: swap 인터페이스 선언 |
| `kernel/swap.c` | **신규**: bitmap + swapout/swapin |
| `kernel/vm.c` | `walk` static 제거 + `uvmcopy`/`uvmunmap` swap PTE 처리 |
| `kernel/trap.c` | `vmfault()` 안에 case 2 추가 (swap된 PTE → swapin) |
| `kernel/kalloc.c` | (선택) `freemem_count()` 외 추가 없음 |
| `kernel/sysproc.c` | `sys_swapout` 신규 |
| `kernel/syscall.h` | `SYS_swapout = 29` |
| `kernel/main.c` | `swapinit()` 호출 추가 |
| `kernel/defs.h` | swap 함수 prototype |
| `Makefile` | swap.img 생성 + qemu -drive 옵션 |

## 6. LLM 통신 프로토콜 (memhint)

### xv6 → bridge
```
<<MEM>>
total=134217728 free=52428800 swap_total=4194304 swap_used=1048576
pid name sz priority
3 spin 8192000 10
4 stressfs 2097152 10
5 sh 65536 10
<<MEM_END>>
```

### bridge → xv6
```json
{"action": "kill",  "pid": 3,  "rationale": "spin using 8MB, memory critical"}
{"action": "quota", "pid": 3,  "bytes": 4194304}
{"action": "swap",  "pid": 3,  "rationale": "swap out spin's idle pages"}
{"action": "noop"}
```

### 안전장치 (`memhint` 클라이언트 + bridge 양쪽)
- `kill`/`quota`/`swap` 모두 pid 1(init), pid 2(sh) 거부
- `quota`는 최소 1MB 이상만 허용
- `swap`은 RUNNING/RUNNABLE 상태 프로세스에만 허용
- 알 수 없는 action은 noop으로 처리

## 7. 구현 단계 (Phase)

### Phase 1 — 관찰 (필수, 1주)
- `proc_info`에 `sz` 추가
- `sys_getmemstat` 구현
- `user/memhint.c` 스냅샷 송신 (액션 적용 없음)
- bridge에 `<<MEM>>` 프로토콜 추가
- **완료 조건**: `memhint` 실행 시 LLM이 메모리 상태 요약 출력 (action은 항상 noop도 OK)

### Phase 2 — 제어 (필수, 1~2주)
- `sys_setmemquota` 구현 (sbrk lazy + uvmalloc 양쪽 체크)
- `mem_quota` 필드 추가
- LLM kill/quota 액션 적용
- **완료 조건**: `spin` 여러 개 실행 시 LLM이 가장 큰 메모리 사용 프로세스에 quota 적용 또는 kill

### Phase 3 — Swap (Optional, 2~4주)
- 위 "선결 조건" 6개 모두 해결한 후 시작
- `kernel/swap.{h,c}` 작성

### Phase 4 — 평가 (필수, 1주)

| 지표 | 측정 방법 | LLM-off vs LLM-on |
|------|----------|-------------------|
| 처리량 | `spin` 워크로드 단위 시간당 iteration | quota로 공정 분배 시 분산 감소 |

```c
uint64 sys_setmemquota(void) {
  int pid;
  uint64 bytes;
  argint(0, &pid);
  argaddr(1, &bytes);

  // 안전장치: init/sh 보호 + 최소 크기
  if(pid == 1 || pid == 2) return -1;
  if(bytes > 0 && bytes < 1024*1024) return -1;  // 1MB 미만 거부

  struct proc *p;
  for(p = proc; p < &proc[NPROC]; p++){
    acquire(&p->lock);
    if(p->pid == pid && p->state != UNUSED){
      p->mem_quota = bytes;
      release(&p->lock);
      return 0;
    }
    release(&p->lock);
  }
  return -1;
}
```

```c
// sys_swapout: LLM 명령으로 특정 프로세스 페이지 스왑
uint64 sys_swapout(void) {
  int pid;
  argint(0, &pid);

  if(pid == 1 || pid == 2) return -1;

  struct proc *p;
  for(p = proc; p < &proc[NPROC]; p++){
    acquire(&p->lock);
    if(p->pid == pid && (p->state == RUNNING || p->state == RUNNABLE
                          || p->state == SLEEPING)){
      int r = swapout_proc(p);  // p->lock을 잡은 채 호출
      release(&p->lock);
      return r;
    }
    release(&p->lock);
  }
  return -1;
}
```

### 10-6. `kernel/swap.c` 핵심 (Phase 3)
```c
#include "types.h"
#include "riscv.h"
#include "defs.h"
#include "spinlock.h"
#include "proc.h"
#include "swap.h"
#include "fs.h"
#include "buf.h"

static struct spinlock swap_lock;
static uint8 swap_bitmap[SWAPSLOTS];

void swapinit(void) {
  initlock(&swap_lock, "swap");
  memset(swap_bitmap, 0, sizeof(swap_bitmap));
}

uint32 swap_alloc(void) {
  acquire(&swap_lock);
  for(int i = 0; i < SWAPSLOTS; i++){
    if(swap_bitmap[i] == 0){
      swap_bitmap[i] = 1;
      release(&swap_lock);
      return i;
    }
  }
  release(&swap_lock);
  return (uint32)-1;
}

void swap_free(uint32 slot) {
  if(slot >= SWAPSLOTS) return;
  acquire(&swap_lock);
  swap_bitmap[slot] = 0;
  release(&swap_lock);
}

uint64 swapused_count(void) {
  uint64 n = 0;
  acquire(&swap_lock);
  for(int i = 0; i < SWAPSLOTS; i++) if(swap_bitmap[i]) n++;
  release(&swap_lock);
  return n;
}

// 페이지 → 디스크 (PTE V=0, PPN 자리에 slot 번호 인코딩)
// 주의: walk()는 vm.c에서 static 제거 또는 walk_pub 래퍼 필요
void swapout_page(pagetable_t pt, uint64 va, uint32 slot) {
  pte_t *pte = walk(pt, va, 0);
  if(!pte || !(*pte & PTE_V)) return;

  uint64 pa = PTE2PA(*pte);
  uint32 blk = slot * (PGSIZE / BSIZE);

  for(int i = 0; i < PGSIZE/BSIZE; i++){
    struct buf *b = bread(SWAPDEV, blk + i);
    memmove(b->data, (char*)pa + i*BSIZE, BSIZE);
    bwrite(b);
    brelse(b);
  }

  // PTE: V=0, PPN 자리에 slot 번호 (lazy 미할당의 *pte==0과 구분됨)
  *pte = (((uint64)slot) << 10);
  kfree((void*)pa);
  sfence_vma();  // TLB invalidate
}

void swapin_page(pagetable_t pt, uint64 va, uint32 slot) {
  char *mem = kalloc();
  if(!mem) panic("swapin: out of memory");

  uint32 blk = slot * (PGSIZE / BSIZE);
  for(int i = 0; i < PGSIZE/BSIZE; i++){
    struct buf *b = bread(SWAPDEV, blk + i);
    memmove(mem + i*BSIZE, b->data, BSIZE);
    brelse(b);
  }

  pte_t *pte = walk(pt, va, 0);
  *pte = PA2PTE((uint64)mem) | PTE_U | PTE_R | PTE_W | PTE_V;
  swap_free(slot);
  sfence_vma();
}

// Clock 알고리즘으로 A=0 페이지 찾아 swapout
// 호출자가 p->lock을 보유한 채로 진입한다고 가정
int swapout_proc(struct proc *p) {
  for(int pass = 0; pass < 2; pass++){
    for(uint64 va = 0; va < p->sz; va += PGSIZE){
      pte_t *pte = walk(p->pagetable, va, 0);
      if(!pte || !(*pte & PTE_V)) continue;
      if(*pte & PTE_A){
        if(pass == 0){
          *pte &= ~PTE_A;  // 두 번째 패스에서 후보가 됨
        }
        continue;
      }
      uint32 slot = swap_alloc();
      if(slot == (uint32)-1) return -1;
      swapout_page(p->pagetable, va, slot);
      return 0;
    }
  }
  return -1;
}
```

### 10-8. `kernel/vm.c` Phase 3 - uvmcopy/uvmunmap swap PTE 처리
```c
// uvmcopy: V=0이지만 swap 마커가 있는 경우 처리
int uvmcopy(pagetable_t old, pagetable_t new, uint64 sz) {
  for(uint64 i = 0; i < sz; i += PGSIZE){
    pte_t *pte;
    if((pte = walk(old, i, 0)) == 0) continue;

    if(!(*pte & PTE_V)){
      // V=0이고 swap된 경우: 새 슬롯에 복제
      if((*pte >> 10) != 0){
        uint32 old_slot = (uint32)(*pte >> 10);
        uint32 new_slot = swap_alloc();
        if(new_slot == (uint32)-1) goto err;
        // 디스크 블록 복사 (간단히 swapin → swapout으로 처리하거나
        // 별도 disk-to-disk copy 구현)
        // ...
        pte_t *npte = walk(new, i, 1);
        *npte = ((uint64)new_slot) << 10;
      }
      continue;  // V=0이고 slot도 0이면 lazy 미할당, 그냥 skip
    }
    // (기존 V=1 페이지 복사 로직)
  }
  return 0;
err:
  uvmunmap(new, 0, i / PGSIZE, 1);
  return -1;
}

// uvmunmap: V=0 swap PTE도 함께 해제
void uvmunmap(pagetable_t pagetable, uint64 va, uint64 npages, int do_free) {
  for(uint64 a = va; a < va + npages*PGSIZE; a += PGSIZE){
    pte_t *pte = walk(pagetable, a, 0);
    if(pte == 0) continue;

    if(!(*pte & PTE_V)){
      if((*pte >> 10) != 0){
        // swap 슬롯 해제
        swap_free((uint32)(*pte >> 10));
      }
      *pte = 0;  // freewalk panic 방지
      continue;
    }
    // (기존 V=1 페이지 해제 로직)
  }
}
```

### 10-9. `user/memhint.c` Phase 1+2 버전
```c
// memhint — 메모리 스냅샷 전송 + LLM 액션 적용
#include "kernel/types.h"
#include "kernel/stat.h"
#include "kernel/memstat.h"
#include "user/user.h"

#define MAX_PROCS 64
#define LINEBUF   256

int main(void) {
  struct memstat ms;
  if(getmemstat(&ms) < 0){
    fprintf(2, "memhint: getmemstat failed\n");
    exit(1);
  }

  static struct proc_info procs[MAX_PROCS];
  int n = proclist(procs, MAX_PROCS);

  // 스냅샷 전송
  printf("<<MEM>>\n");
  printf("total=%lld free=%lld swap_total=%lld swap_used=%lld\n",
         ms.total, ms.free, ms.swap_total, ms.swap_used);
  printf("pid name sz priority\n");
  for(int i = 0; i < n; i++){
    if(procs[i].state == 0) continue;  // UNUSED 스킵
    printf("%d %s %lld %d\n",
           procs[i].pid, procs[i].name,
           procs[i].sz, procs[i].priority);
  }
  printf("<<MEM_END>>\n");

  // bridge 응답 수신 및 적용
  char line[LINEBUF];
  if(gets(line, LINEBUF) == 0){ fprintf(2, "memhint: no reply\n"); exit(1); }
  if(strstr(line, "noop")){ printf("memhint: noop\n"); exit(0); }

  int pid = 0;
  char *p = strstr(line, "\"pid\":");
  if(p) pid = atoi(p + 6);

  // 클라이언트 측 안전장치 (bridge가 깨져도 막음)
  if(pid == 1 || pid == 2){
    printf("memhint: refused (pid %d protected)\n", pid);
    exit(0);
  }

  if(strstr(line, "\"kill\"") && pid > 0){
    kill(pid);
    printf("memhint: killed pid=%d\n", pid);
  } else if(strstr(line, "\"quota\"") && pid > 0){
    char *b = strstr(line, "\"bytes\":");
    uint64 bytes = b ? (uint64)atoi(b + 8) : 4*1024*1024;
    if(bytes < 1024*1024) bytes = 1024*1024;  // 최소 1MB
    setmemquota(pid, bytes);
    printf("memhint: quota pid=%d bytes=%lld\n", pid, bytes);
  } else if(strstr(line, "\"swap\"") && pid > 0){
    // Phase 3에서만 활성화
    // swapout(pid);
    printf("memhint: swap requested (Phase 3 not yet active)\n");
  }
  exit(0);
}
```

### RISC-V 주의사항
- PTE_A (Accessed) 비트: RISC-V 하드웨어가 자동 세팅, 소프트웨어가 클리어 가능
- `sfence.vma` 명령 필요: 페이지 테이블 수정 후 TLB 무효화
  ```c
  asm volatile("sfence.vma zero, zero");
  ```
  또는 xv6의 `sfence_vma()` 헬퍼 호출
