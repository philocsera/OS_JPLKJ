# LLM 기반 메모리 관리 확장 계획

> xv6-riscv에 LLM이 메모리 정책을 제안하고 커널이 실행하는 구조를 추가한다.
> 기존 `schedhint` (CPU 스케줄링 힌트)의 메모리 버전.

---

## 1. 전체 방향

```
xv6 커널 (우리가 직접 구현)          호스트 bridge          LLM (Solar)
─────────────────────────────        ──────────────         ───────────
proc_info에 sz 포함                  <<MEM>> 수신           메모리 압박
sys_getmemstat                  →    Solar API 호출    →    프로세스 분석
sys_setmemquota                 ←    <<MEMACT>> 응답   ←    액션 제안
swap 커널 구현
```

LLM은 직접 메모리를 건드리지 않는다.
커널이 구현한 syscall만 사용하며, LLM은 정책 힌트만 제공한다.

---

## 2. 필요한 이론

### 2-1. xv6 현재 메모리 구조 (기반 지식)

| 파일 | 역할 |
|------|------|
| `kernel/kalloc.c` | 물리 메모리 할당/해제. 4KB 페이지 단위 free list |
| `kernel/vm.c` | 가상 메모리, 페이지 테이블 조작 |
| `kernel/memlayout.h` | 물리 주소 레이아웃 상수 |

- 물리 메모리: QEMU 기본 128MB (`PHYSTOP = 0x88000000`)
- `kalloc()` 실패 시 그냥 0 반환 → 프로세스 죽음
- Swap 없음, OOM 킬러 없음, 메모리 쿼터 없음

### 2-2. RISC-V Sv39 페이지 테이블

xv6-riscv는 Sv39 방식: 가상 주소 39비트, 3단계 페이지 테이블.

PTE (Page Table Entry) 구조:
```
[63:10] PPN (Physical Page Number)
[9]     RSW (소프트웨어용)
[8]     D  — Dirty: 페이지가 쓰여진 적 있음 (하드웨어 세팅)
[7]     A  — Accessed: 페이지가 접근된 적 있음 (하드웨어 세팅)
[6]     G  — Global
[5]     U  — User 접근 가능
[4]     X  — Executable
[3]     W  — Writable
[2]     R  — Readable
[1]     V  — Valid: 0이면 이 PTE로 접근 시 페이지 폴트 발생
```

**핵심**: V=0으로 설정하면 해당 페이지 접근 시 trap이 발생한다.
Swap 구현에서 이 비트를 이용해 "이 페이지는 디스크에 있음"을 표시한다.

### 2-3. 페이지 폴트 처리 (Swap의 핵심)

현재 xv6 `kernel/trap.c`:
```c
// 페이지 폴트 발생 시 그냥 죽임
default:
  ...
  setkilled(p);
```

Swap 구현 후:
```
페이지 폴트 발생 (scause = 13: load, 15: store)
    ↓
stval 레지스터에서 폴트 발생 주소 읽기
    ↓
해당 PTE 확인 → V=0이고 스왑 마커가 있으면?
    ↓
디스크에서 페이지 읽어서 물리 메모리에 올림
    ↓
PTE 복원 (V=1), 프로세스 재개
```

RISC-V에서 페이지 폴트 관련 레지스터:
- `scause`: 폴트 종류 (13=load, 15=store, 12=instruction)
- `stval`: 폴트 발생한 가상 주소

### 2-4. 페이지 교체 알고리즘

메모리가 부족할 때 어떤 페이지를 디스크로 내보낼지 결정.

**Clock 알고리즘 (LRU 근사, xv6에 적합)**
```
페이지들을 원형 리스트로 관리
시계 바늘이 돌면서:
  - A비트 = 1 → 0으로 초기화 후 넘어감 (최근 사용됨)
  - A비트 = 0 → 이 페이지를 교체 대상으로 선택 (오래 안 사용됨)
```

RISC-V PTE의 A(Accessed) 비트를 직접 읽을 수 있어서 구현 가능.

### 2-5. Swap 파티션 설계

xv6는 virtio 디스크 하나를 사용한다.
현재: 전체를 파일시스템 (`fs.img`)으로 사용.

Swap 추가 방법:
- **방법 A**: `fs.img` 크기를 늘리고 뒷부분을 swap 영역으로 예약
- **방법 B**: 별도 virtio 디스크 추가 (`-drive file=swap.img,...`)

방법 A가 Makefile 수정 최소화로 더 간단하다.

Swap 슬롯 관리:
```c
#define SWAPSIZE  (1024)  // 스왑 슬롯 개수 (= 4MB)
uint8 swap_bitmap[SWAPSIZE];  // 0=free, 1=used
```

PTE에 스왑 슬롯 번호 인코딩 (V=0일 때 PPN 자리 재활용):
```
PTE [V=0, PPN = swap_slot_number]
```

## 3. 신규 syscall 설계

### 기존 syscall 수정

**`sys_proclist` (SYS_proclist = 26)**
- `struct proc_info`에 `uint64 sz` 필드 추가
- `proc->sz`를 복사해서 넘겨줌
- 이것만으로 LLM이 프로세스별 메모리 사용량 관찰 가능

### 신규 syscall

| 번호 | 이름 | 인자 | 반환 | 설명 |
|------|------|------|------|------|
| 27 | `sys_getmemstat` | `struct memstat *` | 0/-1 | 전체+스왑 메모리 통계 |
| 28 | `sys_setmemquota` | `pid, uint64 bytes` | 0/-1 | 프로세스 메모리 상한 설정 |
| 29 | `sys_swapout` | `pid` | 0/-1 | 특정 프로세스 페이지 강제 스왑 (LLM 힌트 실행용) |

#### `struct memstat` 정의 (신규, `kernel/memstat.h`)
```c
struct memstat {
  uint64 total;       // 전체 물리 메모리 (bytes)
  uint64 free;        // 현재 여유 메모리
  uint64 swap_total;  // 스왑 총 크기
  uint64 swap_used;   // 스왑 사용 중인 크기
};
```

#### `sys_setmemquota` 동작
```c
// kernel/proc.h의 struct proc에 추가
uint64 mem_quota;  // 0이면 무제한

// kernel/vm.c의 uvmalloc() (sbrk 내부)에서 체크
if(p->mem_quota && p->sz + n > p->mem_quota)
  return 0;  // 할당 거부
```

#### `sys_swapout` 동작
```
1. pid로 프로세스 찾기
2. 해당 프로세스의 페이지 테이블 순회
3. A비트=0인 페이지 (최근 미사용) 찾기
4. 페이지를 디스크에 쓰고 PTE V=0으로
5. 물리 페이지 kalloc free list에 반환
```

## 5. LLM 통신 프로토콜 (memhint)

### xv6 → bridge
```
<<MEM>>
total=134217728 free=52428800 swap_total=4194304 swap_used=1048576
pid name sz priority
3 spin 8192000 10
4 stressfs 2097152 10
5 sh 65536 10
<<MEM_END>>
```

### bridge → xv6
```json
{"action": "kill",  "pid": 3,  "rationale": "spin using 8MB, memory critical"}
{"action": "quota", "pid": 3,  "bytes": 4194304}
{"action": "swap",  "pid": 3,  "rationale": "swap out spin's idle pages"}
{"action": "noop"}
```

### 안전장치
- `kill`은 pid 1(init), pid 2(sh) 불가
- `quota`는 최소 1MB 이상만 허용
- `swap`은 RUNNING 상태 프로세스에만 허용
- delta가 아닌 절대값이므로 별도 clamp 불필요

- **목표**: LLM이 메모리 과다 사용 프로세스를 제어할 수 있음

### Phase 3 — Swap (2~4주)

- **목표**: 물리 메모리 부족 시 자동 스왑, LLM이 스왑 대상 제안

  - 스왑 없을 때 vs 있을 때 프로세스 생존율

## 9. 구체적 구현 코드

### 9-1. `kernel/swap.h`
```c
#define SWAPSLOTS 1024          // 슬롯 수 (× 4KB = 4MB 스왑)
#define SWAP_START_BLOCK 2000   // fs.img에서 스왑 영역 시작 블록 번호

void            swapinit(void);
uint32          swap_alloc(void);           // 빈 슬롯 번호 반환
void            swap_free(uint32 slot);
void            swapout_page(pagetable_t pt, uint64 va, uint32 slot);
void            swapin_page(pagetable_t pt, uint64 va, uint32 slot);
int             swapout_proc(struct proc *p); // 프로세스에서 페이지 하나 evict
```

### 9-2. `kernel/swap.c` 핵심 구조
```c
#include "types.h"
#include "riscv.h"
#include "defs.h"
#include "spinlock.h"
#include "proc.h"
#include "swap.h"
#include "fs.h"
#include "buf.h"

static struct spinlock swap_lock;
static uint8 swap_bitmap[SWAPSLOTS];  // 0=free, 1=used

void swapinit(void) {
  initlock(&swap_lock, "swap");
  memset(swap_bitmap, 0, sizeof(swap_bitmap));
}

uint32 swap_alloc(void) {
  acquire(&swap_lock);
  for(int i = 0; i < SWAPSLOTS; i++){
    if(swap_bitmap[i] == 0){
      swap_bitmap[i] = 1;
      release(&swap_lock);
      return i;
    }
  }
  release(&swap_lock);
  return (uint32)-1;  // 스왑 공간 부족
}

void swap_free(uint32 slot) {
  acquire(&swap_lock);
  swap_bitmap[slot] = 0;
  release(&swap_lock);
}

// 페이지 → 디스크 (PTE V=0, PPN = slot 번호로 표시)
void swapout_page(pagetable_t pt, uint64 va, uint32 slot) {
  pte_t *pte = walk(pt, va, 0);
  if(!pte || !(*pte & PTE_V)) return;

  uint64 pa = PTE2PA(*pte);
  uint32 blk = SWAP_START_BLOCK + slot * (PGSIZE / BSIZE);

  // 페이지를 디스크에 씀 (PGSIZE / BSIZE = 8 블록)
  for(int i = 0; i < PGSIZE/BSIZE; i++){
    struct buf *b = bread(ROOTDEV, blk + i);
    memmove(b->data, (char*)pa + i*BSIZE, BSIZE);
    bwrite(b);
    brelse(b);
  }

  // PTE: V=0, PPN 자리에 slot 번호 인코딩
  *pte = (((uint64)slot) << 10);  // V=0, R/W/X=0
  kfree((void*)pa);               // 물리 페이지 반환
}

// 디스크 → 페이지 (PTE V=1 복원)
void swapin_page(pagetable_t pt, uint64 va, uint32 slot) {
  char *mem = kalloc();
  if(!mem) panic("swapin: out of memory");

  uint32 blk = SWAP_START_BLOCK + slot * (PGSIZE / BSIZE);
  for(int i = 0; i < PGSIZE/BSIZE; i++){
    struct buf *b = bread(ROOTDEV, blk + i);
    memmove(mem + i*BSIZE, b->data, BSIZE);
    brelse(b);
  }

  pte_t *pte = walk(pt, va, 0);
  *pte = PA2PTE((uint64)mem) | PTE_U | PTE_R | PTE_W | PTE_V;
  swap_free(slot);
}

// 프로세스에서 A비트=0인 페이지 하나 스왑아웃
int swapout_proc(struct proc *p) {
  for(uint64 va = 0; va < p->sz; va += PGSIZE){
    pte_t *pte = walk(p->pagetable, va, 0);
    if(!pte || !(*pte & PTE_V)) continue;
    if(*pte & PTE_A){
      *pte &= ~PTE_A;  // A비트 클리어, 다음 라운드에서 교체 대상
      continue;
    }
    uint32 slot = swap_alloc();
    if(slot == (uint32)-1) return -1;
    swapout_page(p->pagetable, va, slot);
    return 0;
  }
  return -1;
}
```

### 9-3. `kernel/trap.c` 페이지 폴트 핸들러 추가
```c
// usertrap() 내부, scause 처리 부분에 추가
// RISC-V: scause 13 = load page fault, 15 = store page fault

uint64 cause = r_scause();
if(cause == 13 || cause == 15){
  uint64 va = PGROUNDDOWN(r_stval());  // 폴트 발생 가상 주소
  pte_t *pte = walk(p->pagetable, va, 0);

  if(pte && !(*pte & PTE_V) && (*pte >> 10) != 0){
    // V=0이고 PPN 자리에 슬롯 번호가 있음 → 스왑된 페이지
    uint32 slot = (uint32)(*pte >> 10);
    swapin_page(p->pagetable, va, slot);
    // 복귀 후 같은 명령어 재실행됨
  } else {
    // 진짜 잘못된 접근
    printf("usertrap: page fault va=%p pid=%d\n", va, p->pid);
    setkilled(p);
  }
}
```

### 9-4. `kernel/sysproc.c` 신규 핸들러 3개
```c
// sys_getmemstat: 메모리 통계를 유저 포인터에 복사
uint64 sys_getmemstat(void) {
  uint64 addr;
  argaddr(0, &addr);
  struct memstat ms;
  ms.total     = (uint64)PHYSTOP - (uint64)end;
  ms.free      = freemem_count() * PGSIZE;   // kalloc.c에서 export
  ms.swap_total = (uint64)SWAPSLOTS * PGSIZE;
  ms.swap_used  = swapused_count() * PGSIZE; // swap.c에서 export
  return copyout(myproc()->pagetable, addr,
                 (char*)&ms, sizeof(ms)) < 0 ? -1 : 0;
}

// sys_setmemquota: 프로세스 메모리 상한 설정
uint64 sys_setmemquota(void) {
  int pid;
  uint64 bytes;
  argint(0, &pid);
  argaddr(1, &bytes);
  struct proc *p = find_proc_by_pid(pid);  // proc.c에 추가 필요
  if(!p) return -1;
  acquire(&p->lock);
  p->mem_quota = bytes;
  release(&p->lock);
  return 0;
}

// sys_swapout: LLM 명령으로 특정 프로세스 페이지 스왑
uint64 sys_swapout(void) {
  int pid;
  argint(0, &pid);
  struct proc *p = find_proc_by_pid(pid);
  if(!p || p->pid == 1 || p->pid == 2) return -1;
  return swapout_proc(p);
}
```

### 9-5. `user/memhint.c` 전체 구조 (schedhint.c 패턴)
```c
// memhint — 메모리 스냅샷 전송 + LLM 액션 적용
#include "kernel/types.h"
#include "kernel/stat.h"
#include "kernel/memstat.h"
#include "user/user.h"

#define MAX_PROCS 64
#define LINEBUF   256

int main(void) {
  struct memstat ms;
  if(getmemstat(&ms) < 0){
    fprintf(2, "memhint: getmemstat failed\n");
    exit(1);
  }

  static struct proc_info procs[MAX_PROCS];
  int n = proclist(procs, MAX_PROCS);

  // 스냅샷 전송
  printf("<<MEM>>\n");
  printf("total=%lld free=%lld swap_total=%lld swap_used=%lld\n",
         ms.total, ms.free, ms.swap_total, ms.swap_used);
  printf("pid name sz priority\n");
  for(int i = 0; i < n; i++){
    if(procs[i].state == 0) continue;  // UNUSED 스킵
    printf("%d %s %lld %d\n",
           procs[i].pid, procs[i].name,
           procs[i].sz, procs[i].priority);
  }
  printf("<<MEM_END>>\n");

  // bridge 응답 수신 및 적용
  char line[LINEBUF];
  if(gets(line, LINEBUF) == 0){ fprintf(2, "memhint: no reply\n"); exit(1); }

  // 파싱: {"action":"kill","pid":7} 형태
  // (xv6에 JSON 파서 없으므로 문자열 직접 파싱)
  if(strstr(line, "noop")){ printf("memhint: noop\n"); exit(0); }

  int pid = 0;
  char *p = strstr(line, "\"pid\":");
  if(p) pid = atoi(p + 6);

  if(strstr(line, "\"kill\"") && pid > 0){
    kill(pid);
    printf("memhint: killed pid=%d\n", pid);
  } else if(strstr(line, "\"quota\"") && pid > 0){
    // bytes 파싱
    char *b = strstr(line, "\"bytes\":");
    uint64 bytes = b ? (uint64)atoi(b + 8) : 4*1024*1024;
    setmemquota(pid, bytes);
    printf("memhint: quota pid=%d bytes=%lld\n", pid, bytes);
  } else if(strstr(line, "\"swap\"") && pid > 0){
    swapout(pid);
    printf("memhint: swapout pid=%d\n", pid);
  }
  exit(0);
}
```

```
You are the memory manager for xv6-riscv, an educational OS.
You receive a snapshot of the system's memory state and must decide
what to do to relieve memory pressure.

Output exactly one JSON object, nothing else:

  {"action": "noop"}
  {"action": "kill",  "pid": <pid>}
  {"action": "quota", "pid": <pid>, "bytes": <bytes>}
  {"action": "swap",  "pid": <pid>}

Rules:
- If free memory > 20% of total: always noop.
- Prefer swap > quota > kill (in order of aggressiveness).
- Never kill or quota pid 1 (init) or pid 2 (sh).
- For quota, set bytes to ~50% of the offending process's current sz.
- Choose the process with the largest sz as the target.
- If swap is used > 80%, prefer kill over swap.
```

```python
    """Resolve a <<MEM>>...<<MEM_END>> dump → kill/quota/swap/noop reply."""

        reply = json.loads(raw)
```
