# xv6 학습 정리

## 1. xv6란 무엇인가

xv6는 MIT에서 만든 교육용 운영체제로, 실제 운영체제(Linux)의 핵심 기능을 단순화하여 구현한 미니 운영체제이다.
약 10,000줄의 C 코드로 구성되어 있으며, 프로세스 관리, 메모리 관리, 파일 시스템 등의 기본 기능을 모두 포함하고 있다.
xv6는 RISC-V 아키텍처 기반으로 동작하며, 일반 PC에서는 직접 실행할 수 없기 때문에 QEMU라는 에뮬레이터를 사용하여 가상 환경에서 실행된다.

구조는 다음과 같다:

```
내 컴퓨터 → QEMU(가상 컴퓨터) → xv6(OS) → 사용자 프로그램
```

---

## 2. xv6 소스코드 구조

xv6의 디렉토리는 크게 두 가지로 나뉜다.

- `kernel/`: 운영체제 핵심 코드 (프로세스, 스케줄링 등)
- `user/`: 사용자 프로그램 (ls, cat, echo 등)

주요 파일의 역할은 다음과 같다:

| 파일 | 역할 |
|------|------|
| `proc.h` | 프로세스 구조체 정의 |
| `proc.c` | 프로세스 생성, 종료, 스케줄링 관리 |
| `syscall.c` | 시스템 콜 번호와 함수 연결 |
| `sysproc.c` | 시스템 콜 실제 구현 |
| `trap.c` | 사용자 → 커널 전환 처리 |
| `swtch.S` | 컨텍스트 스위칭 (어셈블리) |

---

## 3. 프로세스 구조 (struct proc)

xv6에서 프로세스는 `struct proc` 구조체로 표현된다.

주요 구성 요소는 다음과 같다:

- `pid`: 프로세스 식별 번호
- `state`: 현재 상태 (RUNNING, RUNNABLE, SLEEPING 등)
- `parent`: 부모 프로세스
- `pagetable`: 가상 메모리 정보
- `trapframe`: 사용자 상태 저장
- `context`: 커널 상태 저장 (컨텍스트 스위칭용)
- `ofile[]`: 열린 파일 목록
- `cwd`: 현재 디렉토리

즉, `struct proc`는 하나의 프로세스에 대한 모든 정보를 담고 있는 구조이다.

## 4. 프로세스 상태 전이

프로세스는 생성부터 종료까지 다음과 같은 상태 변화를 거친다.

```
UNUSED → RUNNABLE → RUNNING
            ↑            ↓
         wakeup       SLEEPING
                         ↓
                       exit
                         ↓
                       ZOMBIE
```

각 상태의 의미는 다음과 같다:

- `UNUSED`: 아직 사용되지 않는 빈 프로세스 슬롯
- `RUNNABLE`: 실행 준비 완료 상태 (CPU 할당 대기)
- `RUNNING`: CPU를 할당받아 실행 중인 상태
- `SLEEPING`: 특정 이벤트(I/O 등)를 기다리는 상태
- `ZOMBIE`: 실행은 끝났지만 부모 프로세스가 아직 정리하지 않은 상태

---

## 5. 상태 전이 과정 설명

1. `fork()` 호출 시 새로운 프로세스가 생성되며 `RUNNABLE` 상태가 된다.
2. 스케줄러가 `RUNNABLE` 상태의 프로세스를 선택하면 `RUNNING` 상태가 된다.
3. 실행 중인 프로세스는 다음과 같은 이유로 상태가 변할 수 있다:
   - 타이머 인터럽트 발생 → `RUNNABLE`로 돌아감 (CPU 양보)
   - I/O 또는 이벤트 대기 → `SLEEPING` 상태로 전환
4. `SLEEPING` 상태의 프로세스는 조건이 만족되면 `wakeup()`에 의해 `RUNNABLE` 상태로 돌아간다.
5. 프로세스가 작업을 마치면 `exit()`을 호출하고 `ZOMBIE` 상태가 된다.
6. 부모 프로세스가 `wait()`을 호출하면 `ZOMBIE` 상태의 프로세스는 완전히 제거되어 `UNUSED` 상태로 돌아간다.

## 6. 시스템 콜 (System Call)

사용자 프로그램은 커널에 직접 접근할 수 없기 때문에 시스템 콜을 통해 운영체제에 기능을 요청한다.

대표적인 시스템 콜:

- `fork()`: 프로세스 생성
- `exit()`: 프로세스 종료
- `read()`: 입력
- `write()`: 출력

시스템 콜의 실행 흐름은 다음과 같다:

```
사용자 프로그램 → ecall → trap.c → syscall.c → sysproc.c → 실제 함수 실행
```

여기서 `ecall`은 사용자 프로그램이 커널에게 서비스를 요청하는 명령어이다.

## 7. 컨텍스트 스위칭 (Context Switching)

CPU는 한 번에 하나의 프로세스만 실행할 수 있기 때문에, 여러 프로세스를 번갈아 실행해야 한다.
이때 수행되는 과정이 컨텍스트 스위칭이다.

과정:

1. 현재 실행 중인 프로세스의 상태를 저장
2. 다음 실행할 프로세스의 상태를 복원
3. 해당 프로세스를 실행

이 작업은 `swtch.S`에서 수행된다.

---

## 8. 스케줄러 (Scheduler)

xv6는 라운드 로빈 방식의 스케줄링을 사용한다.

동작 방식:
- 모든 프로세스를 순회하면서
- `RUNNABLE` 상태의 프로세스를 찾으면 실행
- 실행 후 다시 다음 프로세스를 탐색

특징:
- 구현이 단순함
- 모든 프로세스에 공평한 CPU 기회 제공
- 우선순위 개념 없음

---

## 9. sleep / wakeup 메커니즘

프로세스가 특정 조건을 기다려야 할 경우 CPU를 낭비하지 않기 위해 sleep 상태로 전환된다.

예:
- 입력 대기
- 데이터 도착 대기

동작 흐름:
- `sleep()`: 프로세스를 `SLEEPING` 상태로 전환
- `wakeup()`: 조건이 만족되면 해당 프로세스를 `RUNNABLE` 상태로 변경

> **중요:** `wakeup()`은 즉시 실행을 의미하지 않고, 실행 가능한 상태로 만드는 것에 불과하다.

---

## 10. 사용자 프로그램 추가 방법

xv6에 새로운 프로그램을 추가하는 과정은 다음과 같다:

1. `user/` 디렉토리에 C 파일 작성
2. `Makefile`의 `UPROGS`에 등록
3. `make qemu`로 빌드 및 실행

예:

```
user/myprogram.c 작성 → Makefile에 추가 → 실행
```

---

## 11. 패치 적용

수업에서 제공되는 patch 파일은 코드 변경 내용을 자동으로 적용하는 기능이다.

사용 방법:
```bash
git apply 파일.patch
make qemu
```

되돌리기:
```bash
git checkout -- 파일명
```

---

## 12. 핵심 정리

xv6는 운영체제의 핵심 개념을 학습하기 위한 교육용 시스템으로,
프로세스 생성, 상태 관리, 스케줄링, 시스템 콜, 컨텍스트 스위칭 등의 동작을 직접 코드로 확인하고 수정할 수 있다.

특히 다음 개념이 중요하다:

- 프로세스 상태 전이
- 시스템 콜 흐름
- 스케줄링 방식
- 컨텍스트 스위칭
- sleep / wakeup 메커니즘

이러한 개념을 이해하면 xv6 기반 프로젝트를 수행하는 데 핵심적인 기반이 된다.
